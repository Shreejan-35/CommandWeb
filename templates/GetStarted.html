INSTALLATION

  1. Initial Installation:-


    2. You can download CommandConsole through Github. You need to head over to the CommandConsole repository, after that click on the green Code button and then you can select either:-

    • Download ZIP
    • Clone

      For cloning, open the git CLI in your local computer, then type this command - git clone https://github.com/Shreejan-35/CommandConsole.git. After that hit enter and it will clone clone the repository and you can use CommandConsole.

  2. Testing in Docker:-

    3. If you want to test the application, you can do it in docker. The steps to execute the application using docker are as follows:-
    1. Type:- docker run -it shreejan35/commandconsole:version(All the version you can see in docker hub, the link is given down)
    2. It will pull the image from the docker hub and execute it.
    3. You can execute all the commands in the home directory.
    4. You may not be able run the update.sh in docker.
    5. Link to the docker hub:- shreejan35/commandconsole


UPDATE

Updating the CommandConsole is very easy. The only thing which you need is Git CLI and that's all. The steps are:-

  1. Double click the update.sh file.
  2. Type y for the question - "Do you have git cli?"
  3. Type your username and password correctly.
It's done.


SETTING UP AND ENVIRONMENT OF COMMANDCONSOLE

It is not that hard to initialize the CommandConsole. Just type a good username within 10 characters and a strong password within 10 characters. After that type help if you want to know what are the functions and commands to be executed.
Characteristics of the interface:-

  • The line before typing the command shows the path where you currently are.
  • On the top of it there is a smiley face - :-)
  • First type the command and hit enter
  • After that according to the command give inputs or it may not take input like - date, sys, etc. commands
  • Type quit to exit the terminal, it will show - BYE BYE but you will not be able to see it as it is really fast
