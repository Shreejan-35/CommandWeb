INSTALLATION

  1. Initial Installation:-

    2. There are two ways to for initial installation of CommandConsole.
    First, you can download CommandConsole through Github. You need to head over to the CommandConsole repository, after that click on the green Code button and then you can select either:-

    • Download ZIP
    • Clone

      For cloning, open the git CLI in your local computer, then type this command - git clone https://github.com/Shreejan-35/CommandConsole.git. After that hit enter and it will clone clone the repository and you can use CommandConsole.

    Second, you can download CommandConsole from the download section. It is more suitable then any others as it locates the files and folders in the accurate and specific positions.
    And it is best for normal people. (There you can get only the latest version).
  2. Testing in Docker:-

    3. If you want to test the application, you can do it in docker. The steps to execute the application using docker are as follows:-
    1. Type:- docker run -it shreejan35/commandconsole:version(All the version you can see in docker hub, the link is given down)
    2. It will pull the image from the docker hub and execute it.
    3. You can execute all the commands in the home directory.
    4. You may not be able run the update.sh in docker.
    5. Link to the docker hub:- shreejan35/commandconsole
